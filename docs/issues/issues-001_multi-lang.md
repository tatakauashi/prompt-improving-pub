## 要件
`Fill in the Details` のプルダウンで `Custom` を選択しても入力できないバグの修正

## 要件の詳細
プルダウンから `Custom` を選択すると、入力欄が表示されるが、1文字入力しただけで入力欄が消えてしまう。
この現象を修正する。

## 要件確認への回答

1. 優先する解決策はどれですか？
- 提案3: セレクトの値を保持（最もシンプル）
カスタム入力時でもセレクトの値を__custom__に保ち、実際の値は別に保存する。

2. カスタム入力後の動作
カスタム値を入力後、再度selectで別のオプションを選択した場合、カスタム値は保持する。

3. カスタム値の再編集

シナリオ
    1. ユーザーがプルダウンから「Custom」を選択
    2. 入力欄に「Tokyo」と入力
    3. 「Apply」ボタンをクリックしてモーダルを閉じる
    4. **再度「プロンプトを編集」ボタンをクリックしてモーダルを開く**

この時にどう表示するか？
→ オプションA（カスタムモードを記憶）
  - 「Custom」の選択状態は保持
  - 入力欄には「Tokyo」が表示される
