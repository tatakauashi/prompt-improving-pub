# 本プロジェクトの目的
Web 画面・その他ツールを問わず、AI に対する指示・プロンプトの精度を高めるための仕組みを提供すること。

## 前提
1. AI との対話やプロンプトの書き方には個人差があり、それがそのまま AI の活用度の差となりうる。
2. AI へのプロンプトは構造化されたものが推奨されるが、人間としてはより日常的な自然言語での入力が好まれる。
3. 人間がプロンプトを打ち込む際にはそれぞれの癖のようなものがあり、誤字だったり、回りくどい文だったり、といったように、プロンプトが AI に正確に意図が伝わりにくい場合がある。
4. 人間が AI に指示をする際であっても、その対象事項や前提条件などに対する人間の理解・分析が十分でない場合がある。

## 実現したいこと（ *前提* の項目の数字と対応）
1. 誰でも AI に伝わりやすいプロンプトを書き、適切な指示を出すことができ、要件定義から設計、コーディング補助、設計書やコードやテストケースなどのレビューを行えるようになること。ゴールの明確化（質問か、要約か、設計か？）や出力形式の指定や提案もサポートしてほしい。
2. 可能な限り自然な言葉で入力した指示を元に、構造化の知識を必要とすることなく **対話形式のUIで** 構造化したプロンプトにできること。その際に資料等の過不足を確認し、必要な資料のアップロードや添付を促されると良い。目的に応じたテンプレートをただ提示するのではなく、必要に応じて対話しながら、もしくはフォーム形式で必要事項を埋めていきながら、人間にも分かりやすく、入力しやすい流れでプロンプトを作成できたら良い。
3. 必ずしもリアルタイムである必要はないが、自然言語として入力した内容の不確かさや、あえて構造化した入力内容の不完全さ、あいまいさ、不整合な内容、といったものを改善することができること。
4. 指示をする前に前提条件を確認したり、指示そのものへの妥当性を検証したい。つまり一発で指示をするのではなく、その指示に対する情報の過不足を確認し、疑問点や提案を挟みたい。

## 実現方法
Google Chrome などのブラウザの拡張機能として実装する。
ブラウザの ChatGPT や Claude の画面からチャットを入力し、それを元に構造化したプロンプトに変換する、もしくはその筋道を示す仕組みを提供する。
構造化したプロンプトへの変換、もしくはその筋道を示すために、クライアントサイドスクリプトから AI にリクエストを送信する仕組みを提供する。
AI への API key はブラウザに安全に保存する。設定により、ブラウザを閉じる際に API key を削除するオプションも提供する。

## 実現イメージ
1. 使用する AI の API key を事前に拡張機能に設定しておく。
2. ブラウザの ChatGPT や Claude の画面からチャットを入力する。
3. 「プロンプトをレビュー」ボタンを押す。
4. クライアントサイドスクリプトで指定した AI にリクエストを送信し、その応答を元にプロンプトの改善提案を表示する。
5. 改善提案を元にブラウザの ChatGPT や Claude の画面から入力したチャットを修正したり、AI が提案した改善されたプロンプトを「この内容を採用」ボタンを押すことで ChatGPT や Claude の画面の入力欄にコピーする。
